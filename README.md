## Как запустить

```bash
   git clone https://github.com/ваш-username/Flutter_lab5.git
```
2. Перейдите в папку проекта:
```bash
   cd photo_of_the_day
```
3. Установите зависимости:
```bash
   flutter pub get
```
4. Запустите приложение в браузере Chrome:
```bash
   flutter run -d chrome
```

## Ответы на вопросы

### 1. Что такое Future<T>? Чем отличается от обычного возвращаемого значения?

Future<T> — это объект-обещание, который вернёт значение типа T в будущем. В отличие от обычного возвращаемого значения, которое доступно сразу, результат Future появляется через некоторое время (после сетевого запроса, чтения файла, таймера). Код не ждёт его завершения и продолжает выполняться, а результат обрабатывается позже через .then() или await.

### 2. Что делает await? Блокирует ли он весь поток выполнения?

await приостанавливает только текущую асинхронную функцию до завершения Future, но не блокирует весь поток выполнения. Остальные части приложения (анимации, другие обработчики) продолжают работать. Именно поэтому интерфейс не зависает — пользователь видит индикатор загрузки и может взаимодействовать с приложением.

### 3. Зачем setState() вызывается дважды в _fetchPhoto()?

Первый setState() в начале метода устанавливает _isLoading = true, сбрасывает _errorMessage и _imageUrl — Flutter сразу перестраивает экран и показывает CircularProgressIndicator. Второй setState() в конце устанавливает _isLoading = false и записывает новый _imageUrl — Flutter прячет индикатор загрузки и показывает фото. Это позволяет пользователю видеть актуальное состояние на каждом этапе.

### 4. Почему кнопке передаётся _fetchPhoto без скобок, а не _fetchPhoto()?

_fetchPhoto без скобок — это передача ссылки на функцию. Она будет вызвана позже, когда пользователь нажмёт кнопку. Если написать _fetchPhoto() со скобками, функция вызовется немедленно при построении виджета, а не при нажатии — это приведёт к бесконечному циклу загрузок и ошибкам.

### 5. Чем Image.network() отличается от Image.asset()?

Image.network() загружает изображение по URL из интернета — требует активного подключения и автоматически кэшируется Flutter. Image.asset() загружает изображение из локальных ресурсов приложения (папка assets) — работает всегда без интернета, файлы должны быть заранее добавлены в проект и прописаны в pubspec.yaml.
